# Code of Conduct

This project expects respectful, technical, and privacy-aware collaboration.

## Expected behavior
- Be respectful, patient, and specific when discussing bugs, code, and design decisions.
- Focus criticism on behavior, implementation, or reproducible issues instead of on people.
- Share enough context to make reports actionable.
- Treat account data, session artifacts, and security-sensitive details carefully.

## Unacceptable behavior
- Harassment, discrimination, intimidation, or personal attacks.
- Publishing other people's account material, cookies, storage-state files, HAR captures, or private logs.
- Posting exploit details or sensitive security findings in public issues without coordination.
- Spam, trolling, or intentionally disruptive behavior.

## Scope
This code of conduct applies in issues, pull requests, discussions, release comments, and other project spaces.

## Enforcement
The maintainer may edit, hide, lock, or remove content that violates this policy and may restrict participation when needed to protect contributors and the project.

## Reporting
Report conduct concerns privately through a contact method listed on the maintainer's GitHub profile. Do not open a public issue for conduct or security reports.
